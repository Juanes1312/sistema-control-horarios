### Nodos de despliegue → Protocolos → RNF → Componentes

| Nodos de Despliegue (Infraestructura) | Protocolos de Comunicación | Requisito No Funcional (RNF) Garantizado | Componentes de Software Alojados |
| :--- | :--- | :--- | :--- |
| **Nodo Cliente: Dispositivo de Usuario** (PC, Móvil, Tablet del Empleado o Admin) | HTTPS (TLS 1.2+): Para garantizar el cifrado de datos desde el navegador hacia el servidor. | RNF-03 (Usabilidad): Acceso multiplataforma mediante navegador web sin instalar software local. <br> RNF-01 (Seguridad): Cifrado de credenciales en tránsito. | Componente UI / Interfaz Gráfica: (HTML5, CSS/Bootstrap 5, JavaScript). Vistas renderizadas del sistema. |
| **Nodo Servidor: Servidor Web / Proxy Inverso** (Ej. Nginx o Apache en la nube) | TCP/IP, HTTP/2: Para la recepción y enrutamiento de las peticiones web. | RNF-04 (Escalabilidad): Capacidad de recibir múltiples peticiones simultáneas durante las horas pico de marcaje. | Componente de Enrutamiento: Balanceador de carga y servidor de archivos estáticos (imágenes, CSS). |
| **Nodo Servidor: Servidor de Aplicaciones** (Entorno Python / Django Web Server) | WSGI / ASGI: Protocolo de comunicación interno entre el servidor web y el framework Python. | RNF-01 (Seguridad y Lógica): Validación de IPs, control de sesiones y ejecución de reglas de negocio. | Módulo Central de Asistencia (Services, Views). <br> Módulo de Autenticación. <br> Motor de Consultas y Reportes. |
| **Nodo Servidor: Servidor de Base de Datos** (Servidor MySQL) | MySQL Wire Protocol (Puerto TCP 3306): Protocolo binario nativo para la conexión entre Django (ORM) y la base de datos. | RNF-02 (Persistencia y Rendimiento): Garantiza que los registros no se pierdan, soporta transacciones ACID y permite consultas rápidas. | Motor MySQL Server. <br> Esquema Relacional sistema_control_horarios (Tablas DDL, Vistas, Procedimientos). |
